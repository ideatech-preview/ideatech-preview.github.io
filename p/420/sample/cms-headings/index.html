CMS統一スタイル プレビュー（記事運用チーム向け）

コラム / ニュース / セミナー / 導入事例 の本文に統一適用される見出し・テキスト・リスト・引用・テーブルの表記サンプル。デザインシステム v3 の CMS プラネットルールに準拠。

2026.05.20
サンプルプレビューデザイン

// SUMMARY

このページは内部用プレビューです。CMS本文の見出し・テキスト・リスト・引用・テーブルが統一スタイルで表示されることを確認できます。

// INDEX

  2. 見出しの階層
  3. h3 中見出し
  4. h4 小見出し

はじめに（h2 / 22px） Bold混じり

通常テキスト（16px）。本文の段落です。太字、イタリック、リンクなどを含む表現も確認できます。日本語の可読性を確かめるため、長め短めの文を混ぜています。

見出しの階層（h2 / 22px） Bold混じり

段落の例：AIが回答を作るときの根拠は、ドメイン信頼度・登場頻度・文脈整合性の3つで決まります。これらを満たすには、第三者メディアでの露出と、自社サイトでの一次情報の発信を両立する必要があります。h1タグは記事タイトルのみ該当するので、記事作成においてはh1タグを使用しないこと。

h3 中見出し（18px） Bold混じり

h3 は左の青3pxボーダーで中見出し感を出す。下に続く本文との関係性を視覚的に示します。

h4 小見出し（17px） Bold混じり

h4 は文字色を青にすることで、小ブロックの頭として機能。登壇者プロフィール、Q&A の質問、ポイント解説などで使う想定。

h3 中見出し（再掲）

同じ h3 を再掲。複数回出てもスタイルがブレないことを確認。

リストの表示（h2 / 22px）

箇条書きリスト（ul）

  • 箇条書きアイテム 1
  • 箇条書きアイテム 2 太字を含む
  • 箇条書きアイテム 3 で改行が入る場合の見え方も確認。長い文章を入れて1行に収まらないケースをチェックします。

番号付きリスト（ol）

  1. 番号付きアイテム 1
  2. 番号付きアイテム 2 太字を含む
  3. 番号付きアイテム 3 で改行が入る場合の見え方も確認。長い文章を入れて1行に収まらないケースをチェックします。

その他のブロック（h2 / 22px）

引用ブロック

引用ブロック。Notion で quote を使うとここに出力されます。強調部分も自然に表示。
