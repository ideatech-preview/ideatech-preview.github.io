// THE PROBLEM

完成したWEBサイトという考え方は、もう終わっている。

多くの企業のWEBサイトは「2〜3年に一度のリニューアル」で止まっている。
前回つくった瞬間からずっと、表面は同じ姿のまま、中身は古くなり続けている。
変化のないサービス・組織・WEBサイトは、どんどん腐ってゆく。

IDEA DESIGN®は、変化し続けることを普遍とするデザインを推進し、
常に新しく生まれ変わるWEBサイトの運用を目指します。

SIGN_01

1ヶ月以上、サイトを更新していない

新しい話題に触れていない・事業の進化が反映されていないサイトは、本来の役割として機能しません。
訪れたユーザーから「動いていない会社」だと判断され、AIからも最新性の低い情報源として参照されにくくなります。

SIGN_02

LLMO / AEOを意識していない

AIに選ばれるための戦略は目まぐるしく変化しています。
構造化データや定義文を整えていないサイトは、ChatGPT・Gemini・Perplexityの回答に引用されず、気づけば競合だけが推薦され続ける状態に陥ります。

SIGN_03

コンテンツが不足している

情報発信のネタがないと、企業はどんどん埋もれていきます。
単なるブログ更新ではなく、独自調査やレポートを起点に「AIに引用される一次情報」を継続的に積み上げる仕組みが必要です。

// THE CONCEPT

【動的平衡（どうてきへいこう）】とは、
変わり続けることで、崩れない状態を保つこと。

生命が細胞を絶えず入れ替えながら、常にフレッシュな自分の輪郭を保ち続けるように。
動的平衡WEBサイトは、流動的に情報が更新され続け、逆説的に芯（コアなアイデンティティ）だけが残る、新しいWEBの考え方です。

PARADIGM

デザインを「固める」時代は終わった。

ブランドガイドラインを完璧に固めて納品し、3年は触らない ── そんなデザインは、社会と市場の流動性が一定の閾値を超えた今、固めた瞬間に腐りはじめる。
これからのデザインは流動的に「変わり続ける」ことが基本となる。

PRINCIPLE

「変わらない芯」が浮き彫りになる。

変わり続けるからこそ、自分たちの「ずっと変わらない芯」が浮き彫りになる。
企業の存在意義、顧客との関わり方、人としての在り方….。
己に対する解像度を上げることで、強みを知り、弱点を克服することにつながる。

// MANIFESTO

デザインの常識を覆す、新時代への挑戦

CHALLENGE

デザインガイドラインをあえて「定め切らない」

ガイドラインを定めることは、ブランドイメージを守る一方で、変化への柔軟性や汎用性を下げることとトレードオフです。
また、AIとの共創の可能性に「フタをする足枷」にもなります。
IDEA DESIGN®では、定めるガイドラインを「最低限度」に抑え、もっとも効果的な最新のデザインを探究し続けるスタイルに振り切ります。
ただし、AIに任せきりにはしません。最終的なアウトプットのクオリティは、プロのデザイナーの目で必ず担保します。

// THREE LAYERS

「動的平衡WEBサイト」は、3つのレイヤーで構成される。

「常に最新を発信する」表層、「AIに正確に届く器をつくる」中間、「変化を当たり前にする」深層。
動的平衡WEBサイトは、この3つのレイヤーが連動してはじめて機能します。
1つでも欠ければ、サイトはまた、腐りはじめる。

01

LAYER 01｜SURFACE（表層）

常に最新を発信し続ける「コンテンツ」レイヤー

商品やサービスの情報を常にフレッシュに保ち、新しい発信を継続的に追加する。
サイトを「動いている状態」に保ち続ける、最も表面の層。

  • サービス情報の最新化／新規コンテンツの継続追加
  • 1週間単位の定例MTGで見直し、即サイト反映
  • 社内会議音源 → AI処理 → サイト自動更新

→ AIに「動いている会社」と認識され続ける

02

LAYER 02｜MIDDLE（中間）

AIの目に見つかる「構造化」レイヤー

LLMO / AEO を前提にしたサイト設計と構築。
AIが正確に読み取れる構造化テキスト、定義文、引用フォーマットを器として用意する中間の層。

  • LLMO / AEO を前提にしたサイト設計と構築
  • Schema.org / 構造化テキスト / 定義文最適化
  • 必要に応じて、大胆なリニューアルも積極的に

→ AIから引用される「器」をつくる

03

LAYER 03｜DEEP（最深)

パラダイムを変える「思想・体制」レイヤー

サイトを動的平衡に保ち続けるためには、組織側のカルチャーと意思決定体制ごと変える必要がある。
最も深いところで、組織を動的平衡にする層。

  • 経営／広報チームへの思想インストール
  • 「定め切らない」運用カルチャーの構築
  • 高速PDCAを回し続けるための意思決定体制

→ サイトを基点に、組織やサービスが変化し続ける

// HIDDEN INPUT

死んでいた議事録こそ、サイト更新の「タネ」になる。

ミーティングが終わると、議事録は書庫の奥で眠りはじめる。
書いた人さえ、二度と読み返さない。
でもそこには、事業の最前線・顧客の声・チームの気づきが、すべて詰まっている。

動的平衡WEBサイトは、週次の議事録をAIで読み込み、「今週サイトに反映すべき項目」へ自動的に翻訳。
議事録を書く、という日々の業務が、そのままサイトを育てる行為に変わります。

→眠っていた情報資産が、サイトの鮮度を育てる燃料に変わる。

// FAQ

よくあるご質問

「動的平衡WEBサイト」と一般的なWeb制作会社の違いは何ですか？+
一般的なWeb制作会社は「制作」と「保守（軽微な修正）」が中心で、数年に一度のリニューアルを想定した固定的な関わり方です。動的平衡WEBサイトは、AEO対策を徹底した制作に加え、毎週単位のアップデート運用と、変化を前提とした思想・体制構築までをひとつの商品として束ねます。
毎週本当に変えるべきことがあるのですか？+
事業を真剣にやっている限り、必ずあります。新しい問い合わせ、よくある質問の更新、業界の動き、競合の発信、社内の事例、AIで参照されたキーワードの変化 ── これらすべてが「変えるべき項目」の素材になります。
AEO対策とは具体的に何をしますか？+
Schema.org JSON-LDの設計、構造化テキスト、定義文の最適化、AIに引用されやすい情報密度の設計、独自データ・一次情報の組み込みなど、AIが正確かつ優先的に参照できる形にサイトを設計します。
更新作業は誰がやりますか？社内で運用できますか？+
基本はIDEATECH側で更新作業を担当します。週次MTGで貴社から「変えたい項目」をいただき、こちらでAIと協業して最短即日でリリースします。また、リスクをご理解いただいた上でしたら、編集権限をお渡しすることも可能です。
どんな企業に向いていますか？+
AI時代に「選ばれ続ける状態」をWEBサイトからダイナミックに設計・構築したい中小企業の経営者・広報・マーケティング責任者に向いています。逆に、大手・エンタープライズなど、既存のブランドイメージが強固な企業には向いていません。
契約期間や解約条件を教えてください+
運用ループの効果が現れるまでに最低3ヶ月程度必要なため、初回契約は6ヶ月〜を推奨しています。詳細は事業の状況をお伺いしたうえで個別に設計します。
